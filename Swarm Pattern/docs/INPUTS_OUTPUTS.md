# 📥 INPUTS & OUTPUTS SPECIFICATION

## Overview

This document details all inputs required and outputs generated by the Portfolio Swarm System.

---

## 📥 SYSTEM INPUTS

### 1. **Portfolio Data** (REQUIRED)

#### Position Data (Per Stock)
```python
{
    "ticker": str,              # Stock symbol (e.g., "AAPL")
    "shares": int,              # Number of shares owned
    "current_price": float,     # Current market price per share
    "cost_basis": float,        # Original purchase price per share
    "acquisition_date": date,   # When purchased (for tax calculations)
    "sector": str,              # Sector classification
    "esg_score": int,           # ESG rating (0-100)
    "beta": float               # Market sensitivity (typically 0.5-2.0)
}
```

**Example:**
```python
Position(
    ticker="AAPL",
    shares=1000,
    current_price=150.00,
    cost_basis=140.00,
    acquisition_date="2023-01-15",
    sector="Technology",
    esg_score=72,
    beta=1.2
)
```

#### Portfolio-Level Data
```python
{
    "positions": List[Position],        # All holdings
    "cash": float,                      # Cash balance
    "policy_limits": Dict[str, float]   # Constraints (e.g., max sector %)
}
```

**Example:**
```python
Portfolio(
    positions=[...],  # List of Position objects
    cash=3000000.00,  # $3M cash
    policy_limits={
        "technology_limit": 30.0,      # Max 30% in tech
        "financials_limit": 25.0,      # Max 25% in financials
        "esg_minimum": 60              # Minimum ESG score
    }
)
```

---

### 2. **Swarm Configuration** (OPTIONAL - Has Defaults)

```python
{
    "max_iterations": int,          # Maximum debate rounds (default: 10)
    "consensus_threshold": float,   # Min approval rate (default: 0.6 = 60%)
    "require_unanimous": bool       # All agents must agree (default: False)
}
```

**Example:**
```python
SwarmOrchestrator(
    agents=agents,
    max_iterations=10,
    consensus_threshold=0.6,
    require_unanimous=False
)
```

---

### 3. **Agent Selection** (OPTIONAL - All Active by Default)

```python
{
    "use_market_agent": bool,       # Market analysis (default: True)
    "use_risk_agent": bool,         # Risk assessment (default: True)
    "use_tax_agent": bool,          # Tax optimization (default: True)
    "use_esg_agent": bool,          # ESG compliance (default: True)
    "use_trading_agent": bool       # Execution analysis (default: True)
}
```

---

### 4. **User Preferences** (OPTIONAL - For Production)

```python
{
    "tax_preference": str,          # "minimize_tax" | "maximize_longterm" | "neutral"
    "risk_tolerance": str,          # "conservative" | "moderate" | "aggressive"
    "esg_priority": str,            # "strict" | "moderate" | "flexible"
    "rebalancing_urgency": str,     # "immediate" | "flexible" | "optimize_timing"
    "notification_email": str       # Email for results
}
```

---

## 📤 SYSTEM OUTPUTS

### 1. **Consensus Result** (Primary Output)

```python
{
    "approved": bool,               # Was consensus reached?
    "approval_rate": float,         # Percentage of agents that approved (0.0-1.0)
    "iteration": int,               # Which iteration consensus was reached
    "unanimous": bool,              # Did all agents approve?
    "trade_plan": TradePlan         # The approved trade plan (if consensus)
}
```

**Example:**
```json
{
    "approved": true,
    "approval_rate": 0.8,           # 80% approval (4 of 5 agents)
    "iteration": 1,
    "unanimous": false,
    "trade_plan": { ... }
}
```

---

### 2. **Trade Plan** (If Consensus Achieved)

```python
{
    "trades": List[Trade],                          # Individual trades
    "expected_tax_liability": float,                # Total tax ($)
    "expected_execution_cost": float,               # Brokerage fees ($)
    "execution_timeline_days": int,                 # Days to execute
    "projected_portfolio_metrics": Dict[str, float] # Post-trade metrics
}
```

**Trade Object:**
```python
{
    "action": str,              # "BUY" or "SELL"
    "ticker": str,              # Stock symbol
    "shares": int,              # Number of shares
    "estimated_price": float,   # Expected execution price
    "rationale": str            # Why this trade
}
```

**Example Trade Plan:**
```json
{
    "trades": [
        {
            "action": "SELL",
            "ticker": "NVDA",
            "shares": 2149,
            "estimated_price": 500.00,
```

## 🔄 DATA FLOW SUMMARY

### Input Flow
```
User Input
    ↓
Portfolio Data (positions, cash, limits)
    ↓
Swarm Configuration (iterations, threshold)
    ↓
Agent Selection (which agents to use)
    ↓
User Preferences (optional: tax, risk, ESG priorities)
```

### Output Flow
```
Agent Analysis
    ↓
Debate Messages
    ↓
Trade Proposals
    ↓
Agent Votes
    ↓
Consensus Result
    ↓
Trade Plan (if consensus)
    ↓
Performance Metrics
    ↓
Full Transcript
```

## ✅ VALIDATION RULES

### Input Validation
- ✅ Ticker must be valid string (1-5 characters)
- ✅ Shares must be positive integer
- ✅ Prices must be positive floats
- ✅ Dates must be in past
- ✅ ESG score between 0-100
- ✅ Beta typically 0.1-3.0
- ✅ Consensus threshold between 0.0-1.0

### Output Guarantees
- ✅ Always returns ConsensusResult (even if no consensus)
- ✅ Trade plan only present if consensus achieved
- ✅ All votes accounted for (one per agent)
- ✅ Complete audit trail in messages
- ✅ Performance metrics always calculated
